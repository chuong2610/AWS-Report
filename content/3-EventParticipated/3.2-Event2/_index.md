## 3. In-Depth Content

### 3.1. Prompt Engineering & Foundation Models: Solid Foundation

The event emphasized the principle "Good input creates good output." To work effectively with Foundation Models on Bedrock, mastering these techniques is essential:

* **Zero-shot / Few-shot Prompting:** Techniques for providing context or sample examples to guide AI toward correctly formatted desired results.
* **Chain of Thought (CoT):** Technique requiring AI to "think step-by-step," helping solve complex logical reasoning problems and minimize errors.

### 3.2. AWS AI Services Ecosystem (Pretrained)

API solutions enabling developers to quickly integrate intelligent features into applications:

* **Vision:** *Amazon Rekognition* (Image/video analysis, content moderation).
* **Language:** *Amazon Translate* (Translation), *Comprehend* (Semantic analysis), *Textract* (Document text extraction).
* **Audio:** *Amazon Polly* (Text-to-Speech) and *Transcribe* (Speech-to-Text).

### 3.3. RAG - Solution for Enterprise Data

RAG is the key to enabling AI to understand internal data without expensive fine-tuning:

* **Mechanism:** Uses *Amazon Titan Text Embeddings V2* for data vectorization, enabling semantic search.
* **Knowledge Bases for Amazon Bedrock:** Fully managed service handling the complete RAG workflow: from document chunking, vector storage, retrieval to answer generation.

### 3.4. Advancing to Agentic AI (Task-Oriented AI)

The GenAI trend is strongly shifting from "Question-Answer" to "Action":

* **Hierarchy:** From *Assistant* (single tasks) → *Agents* (goal-oriented, tool-using) → *Agentic Systems* (multi-agent, autonomous).
* **"Prototype to Production" Challenge:** Many Agent projects fail in production due to processing speed issues, data security risks, and poor memory/context maintenance.

### 3.5. Amazon Bedrock AgentCore: Infrastructure for AI Agents

AgentCore solves critical challenges enabling enterprises to confidently deploy Agents:

* **Control:** Strict identity and permission management.
* **Execution Capability:** Integrated *Code Interpreter* (secure Python sandbox) for precise computation.
* **Observability:** Enables tracking Agent reasoning flows for debugging and optimization.
* **Memory:** Maintains context across extended conversation sessions.

### 3.6. Pipecat Framework: The Future of Voice AI

Open-source solution for real-time multimodal communication applications:

* **Advantages:** Extremely low latency creating natural conversation experiences.
* **Pipeline Process:** Parallel processing of WebRTC → STT → LLM → TTS streams ensuring instant responses.

Through this workshop, I systematized the technology development roadmap and extracted key insights:
